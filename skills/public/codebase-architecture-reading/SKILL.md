# Codebase Architecture Reading Skill (分析模式)

## 目的

当用户要求：**"阅读这个项目代码，给我详细讲解"** 时，或用户提示中包含 **"分析模式"** 关键词时，使用此技能。

该技能系统性地：
1. 阅读目标目录的**所有相关代码文件**
2. 理解每个函数的**入参、出参、调用关系**
3. 绘制**完整的流程图**
4. 整理成**小白友好的超详细 markdown 文档**，包含：
   - 文件结构概览
   - 每个模块的详细解析
   - 核心函数的入参出参示例
   - 完整的调用流程图（mermaid）
   - 数据结构定义
   - 真实场景的数据流示例
   - 关键代码摘录 + 逐行分析

## 工作流程

### 第一步：探索代码结构

```
1. 列出目标目录的文件结构（Glob）
2. 读取所有相关的代码文件（Read）
3. 读取相关的测试文件，理解使用方式
4. 识别关键组件的依赖关系
```

**必须读取的文件类型：**
- 目标目录下的所有 `.py` 文件
- 相关的 `test_*.py` 测试文件
- 相关的 `__init__.py` 导出文件
- 相关的 middleware 或调用方文件

### 第二步：理解数据流向

对于每个模块，回答以下问题：
1. **入口在哪里**？谁调用了这个模块？
2. **数据结构是什么**？输入输出的 JSON/dataclass 长什么样？
3. **调用链是什么**？函数 A 调 B，B 调 C... 完整路径是？
4. **关键设计点**？为什么这样设计？有什么好处？

### 第三步：绘制调用流程图

使用 mermaid 绘制：
1. **整体架构图** - 组件之间的关系
2. **时序图** - 函数调用的先后顺序
3. **数据流图** - 数据如何从输入到输出

### 第四步：整理输出结构（超详细）

输出文档必须包含以下部分：

```
# [模块名] 详细解析

## 一、整体架构概览

### 1.1 文件结构
列出所有相关文件及其作用

### 1.2 核心组件关系图
mermaid 图展示组件关系

## 二、完整调用流程图

### 2.1 主流程时序图
mermaid 时序图

### 2.2 函数调用关系速查表
表格形式：函数名 → 调用的函数 → 被谁调用

## 三、[文件 1] 详细解析

### 3.1 概览
这个文件的作用是什么

### 3.2 数据结构
定义的 dataclass/dict 结构，带示例

### 3.3 函数详解

#### 函数名 1
- **作用**：一句话说明
- **入参**：
  ```python
  param1: type  # 说明
  param2: type  # 说明
  ```
- **出参**：
  ```python
  return_type  # 说明 + 示例值
  ```
- **调用关系**：谁调它，它调谁
- **代码分析**：
  - 关键代码摘录（注明文件路径行号）
  - 逐行或逐段解释
  - 设计亮点

#### 函数名 2
...（同上）

## 四、[文件 2] 详细解析
...（同上）

## 五、完整数据流示例

### 5.1 场景描述
用一个真实场景举例

### 5.2 步骤详解
一步步走一遍流程，展示：
- 每一步的输入数据
- 每一步的输出数据
- 数据如何变化

### 5.3 最终结果
展示最终产生的数据/文件内容

## 六、关键设计点总结

1. 设计点 1
2. 设计点 2
...

## 七、文件位置速查表
所有相关文件的路径
```

## 详细要求

### 对小白绝对友好

- **从零开始解释**：假设读者是 Python/AI Agent 小白
- **避免术语堆砌**：如果用了术语，必须解释
- **入参出参必须给示例**：不要只写类型，要写真实的示例值
- **用表格整理**：复杂的关系用表格展示
- **流程图必须有**：mermaid 图比文字好理解 10 倍

### 代码摘录规范

- **摘录要完整**：不要只摘一两行，要能看出上下文
- **注明文件路径和行号**：方便读者定位
- **必须加分析**：不要只贴代码，要解释代码在做什么
- **highlight 关键点**：用注释或单独的分析段落指出重点

### 数据结构示例规范

对于每个重要的 dict/dataclass：
1. 先给**定义**（字段说明）
2. 再给**真实示例**（有值的 JSON）
3. 如果复杂，给**字段说明表格**

### 输出位置

输出文档保存在项目文档目录，例如：
- `backend/docs/MODULE_NAME_DETAILED.md`
- `backend/docs/[模块名]_详细解析.md`
